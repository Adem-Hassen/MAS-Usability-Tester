\chapter{Schema Definitions}
\label{app:schemas}

This appendix presents the key Pydantic data models that define the typed interfaces between pipeline stages. All schemas are defined in the \filepath{schemas/} directory.


% ---- C.1 Persona Schemas ----------------------------------------------------
\section{Persona Schemas}
\label{app:persona-schemas}

\subsection{UIAnalysis}

The \code{UIAnalysis} schema defines the structured output of the supervisor's HTML analysis. It captures the inferred purpose and type of the interface, the critical user paths, all interactive elements, static accessibility risk observations, and an overall risk level.

\begin{lstlisting}[caption={UIAnalysis Pydantic model (from \filepath{schemas/persona\_schema.py}).}, label={lst:uianalysis}]
class UIAnalysis(BaseModel):
    ui_purpose: str          # Inferred purpose of this UI
    ui_type: str             # e.g. 'login form', 'dashboard'
    critical_paths: list[CriticalPath]
    interactive_elements: list[InteractiveElement]
    detected_issues_hint: list[str]   # Static observations
    accessibility_risk_level: Literal["low", "medium", "high"]
\end{lstlisting}

Each \code{InteractiveElement} contains the HTML tag, best CSS selector, visible label, input type, accessibility status, and optional computed styles. Each \code{CriticalPath} defines a named workflow with ordered steps and an accessibility-sensitive flag.

\subsection{PersonaProfile}

The \code{PersonaProfile} schema defines a simulated user persona. It is generated by the supervisor agent and consumed by the persona simulation nodes.

\begin{lstlisting}[caption={PersonaProfile Pydantic model (from \filepath{schemas/persona\_schema.py}).}, label={lst:personaprofile}]
class PersonaProfile(BaseModel):
    persona_id: str               # UUID-based unique identifier
    name: str                     # e.g. 'Elderly screen reader user'
    age_range: str                # e.g. '60-75'
    technical_skill: Literal["low", "medium", "high"]
    accessibility_constraints: list[str]  # e.g. ['uses screen reader']
    cognitive_limitations: list[str]
    task_goal: str                # Specific, actionable goal
    task_context: str             # Motivation and prior context
    entry_point: Optional[str]    # CSS selector or None
    success_criteria: list[str]   # Observable completion conditions
    selection_rationale: str      # Why this persona was generated
    risk_tolerance: Literal["low", "medium", "high"]
    latency_tolerance: Literal["low", "medium", "high"]
    interaction_style: Literal["methodical", "impatient",
                               "exploratory", "cautious"]
\end{lstlisting}


% ---- C.2 Issue Schemas -------------------------------------------------------
\section{Issue Schemas}
\label{app:issue-schemas}

\subsection{IssueReport}

A single usability or accessibility issue found during persona simulation. Produced by the persona agent, consumed by the clustering node.

\begin{lstlisting}[caption={IssueReport Pydantic model (from \filepath{schemas/issue\_schema.py}).}, label={lst:issuereport}]
class IssueReport(BaseModel):
    issue_id: str
    persona_id: str
    UI_page: str
    persona_name: str
    severity: IssueSeverity       # critical | high | medium | low
    category: IssueCategory       # usability | accessibility | ...
    wcag_criterion: Optional[str] # e.g. '1.1.1 Non-text Content'
    title: str
    description: str
    affected_element: Optional[str]       # CSS selector
    affected_element_html: Optional[str]  # Raw HTML snippet
    step_number: int
    page_context: str
    reproduction_steps: list[str]
    persona_impact: str
\end{lstlisting}

\subsection{IssueCluster}

A group of semantically related issues produced by the clustering node. One \code{IssueCluster} maps to one recommender agent instance.

\begin{lstlisting}[caption={IssueCluster Pydantic model (from \filepath{schemas/issue\_schema.py}).}, label={lst:issuecluster}]
class IssueCluster(BaseModel):
    cluster_id: str
    cluster_label: str
    issues: list[IssueReport]
    dominant_category: IssueCategory
    dominant_severity: IssueSeverity
    affected_personas: list[str]
    affected_elements: list[str]
    representative_description: str
    issue_count: int
\end{lstlisting}

\subsection{RecommenderProfile}

Profile for a single recommender agent, generated by the supervisor after clustering.

\begin{lstlisting}[caption={RecommenderProfile Pydantic model (from \filepath{schemas/issue\_schema.py}).}, label={lst:recommenderprofile}]
class RecommenderProfile(BaseModel):
    recommender_id: str
    recommender_name: str         # e.g. 'AriaFixer', 'FormGuard'
    cluster_id: str
    cluster_label: str
    focus: RecommenderFocus       # accessibility | usability | ...
    num_recommenders: int         # 1-4 instances per cluster
    cluster_summary: str
    dominant_severity: str
    affected_elements: list[str]
    wcag_references: list[str]
    fix_strategy_hint: str
    priority: int                 # 1 = highest priority
\end{lstlisting}


% ---- C.3 Patch Schemas -------------------------------------------------------
\section{Patch Schemas}
\label{app:patch-schemas}

\subsection{PatchProposal}

A single patch proposal generated by a recommender agent.

\begin{lstlisting}[caption={PatchProposal Pydantic model (from \filepath{schemas/patch\_schema.py}).}, label={lst:patchproposal}]
class PatchType(str, Enum):
    HTML_ATTRIBUTE   = "html_attribute"
    HTML_STRUCTURE   = "html_structure"
    CONTENT          = "content"
    REMOVE_ELEMENT   = "remove_element"
    REORDER_ELEMENTS = "reorder_elements"
    INLINE_STYLE     = "inline_style"
    CSS_CLASS        = "css_class"
    CSS_RULE         = "css_rule"
    JS_SNIPPET       = "js_snippet"

class PatchProposal(BaseModel):
    patch_id: str
    cluster_id: str
    recommender_id: str
    patch_type: PatchType
    severity_addressed: str
    target_element: str           # CSS selector of target
    description: str
    before_snippet: Optional[str] # Original HTML (for HTML patches)
    after_snippet: Optional[str]  # Fixed HTML (for HTML patches)
    css_snippet: Optional[str]    # CSS rule(s) to inject
    js_snippet: Optional[str]     # JS block to inject before </body>
    confidence: float             # 0.0 to 1.0
    wcag_reference: Optional[str]
    rationale: str
    side_effects: list[str]
\end{lstlisting}

\subsection{ConflictRecord and NegotiationSession}

\begin{lstlisting}[caption={Conflict and negotiation schemas (from \filepath{schemas/patch\_schema.py}).}, label={lst:conflict}]
class ConflictRecord(BaseModel):
    conflict_id: str
    patch_id_a: str
    patch_id_b: str
    target_element: str
    conflict_description: str
    conflict_severity: str

class NegotiationRound(BaseModel):
    round_number: int
    conflict_id: str
    agent_a_argument: str
    agent_b_argument: str
    mediator_assessment: str
    resolution_reached: bool
    proposed_resolution: Optional[str]

class NegotiationSession(BaseModel):
    session_id: str
    conflict: ConflictRecord
    rounds: list[NegotiationRound]
    final_resolution: str         # chose_a | chose_b | merged
    winning_patch_id: Optional[str]
    merged_snippet: Optional[str]
\end{lstlisting}

\subsection{UnifiedPatchSet}

The final aggregated patch set output by the conflict resolution stage.

\begin{lstlisting}[caption={UnifiedPatchSet Pydantic model (from \filepath{schemas/patch\_schema.py}).}, label={lst:unifiedpatchset}]
class UnifiedPatchSet(BaseModel):
    patches: list[ResolvedPatch]
    conflicts_detected: int
    conflicts_resolved: int
    negotiation_sessions: list[NegotiationSession]
    unresolved_conflicts: list[ConflictRecord]
\end{lstlisting}


% ---- C.4 Report Schemas -----------------------------------------------------
\section{Report Schemas}
\label{app:report-schemas}

\subsection{VerificationResult}

Result of re-running persona simulations on the patched HTML.

\begin{lstlisting}[caption={VerificationResult Pydantic model (from \filepath{schemas/report\_schema.py}).}, label={lst:verificationresult}]
class VerificationResult(BaseModel):
    persona_id: str
    persona_name: str
    issues_before: list[str]           # issue_ids before patching
    issues_remaining: list[str]        # issue_ids still present
    issues_resolved: list[str]         # issue_ids successfully fixed
    new_issues_introduced: list[str]   # regression issue_ids
    task_completed_after_patch: bool
    fully_resolved: bool               # All critical/high resolved?
\end{lstlisting}

\subsection{DiagnosticReport}

The final output of the full UI evaluation pipeline.

\begin{lstlisting}[caption={DiagnosticReport Pydantic model (from \filepath{schemas/report\_schema.py}).}, label={lst:diagnosticreport}]
class DiagnosticReport(BaseModel):
    report_id: str
    timestamp: datetime
    html_source_path: str
    ui_context: str
    correction_loop_count: int
    ui_analysis: UIAnalysis
    personas: list[PersonaProfile]
    persona_results: list[PersonaSimulationResult]
    issue_clusters: list[IssueCluster]
    total_issues_found: int
    severity_breakdown: SeverityBreakdown   # critical/high/med/low
    unified_patch_set: UnifiedPatchSet
    total_patches_applied: int
    verification_results: list[VerificationResult]
    issues_resolved_count: int
    issues_remaining_count: int
    regressions_introduced: int
    verification_passed: bool
    overall_score: float        # 0.0 to 10.0
    executive_summary: str
    top_recommendations: list[str]  # Top 5 prioritised improvements
    audit_results: list             # Proactive axe-core audit
\end{lstlisting}
